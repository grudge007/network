Security Mechanisms

  • Access Control Lists (ACLs): Define rules to allow or deny traffic based on IP addresses, protocols, and ports. Implement on core switch and distribution switches to control inter-VLAN communication.
  • VLANs: Segment the network into VLANs for different departments. This limits broadcast domains and restricts access to resources within each VLAN. Configure VLANs on core switch and distribution switches.
  • Port Security: Limit the number of devices that can connect to a port and use MAC address filtering to allow only known devices. Configure on distribution switches.
  • Network Access Control (NAC): Enforce security policies on devices before they can access the network. Implement NAC with a central authentication server connected to the core switch.
  • 802.1X Authentication: Require devices to authenticate before accessing the network using a RADIUS server. Configure 802.1X on core switch and distribution switches.
  • Strong Passwords and Secure Management Interfaces: Use complex passwords and secure management protocols (e.g., SSH, HTTPS). Ensure all switches and routers are configured with strong passwords and management interfaces are secured.
  • Regular Updates and Patching: Keep firmware and software up-to-date to protect against vulnerabilities. Regularly update all network devices, including core switch, distribution switches, and router.
  • Intrusion Detection and Prevention Systems (IDS/IPS): Monitor network traffic for suspicious activity and prevent potential attacks. Deploy IDS/IPS on the core switch or as a separate device in the network.
  • Logging and Monitoring: Enable logging and use centralized monitoring tools to track access and detect anomalies. Configure logging on all network devices and use a centralized monitoring system.
  • Network Segmentation: Further segment the network with additional firewalls or security appliances between VLANs. Consider adding a firewall between VLANs for additional security.
